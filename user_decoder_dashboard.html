01 / FRAMEWORK

三维人格解码框架 · 新增 v3.0

v3.0 框架在原有 ABCD 分层 + 需求双轨（显性/隐性）基础上，新增人格三维速描板块。三套模型分工明确，共同驱动班主任话术从「千人一面」升级为「精准对话」。

🎭
DISC 沟通风格
识别用户的行为腔调——
D型：直接结果导向，用数字和稀缺性打动
I型：热情社交驱动，用故事和情感连接
S型：稳健重承诺，给稳定感不催促
C型：逻辑细节敏感，用数据和流程说服
🔗
依恋类型模型
识别用户对班主任/平台的依附模式——
🟢 安全型：自驱力强，留存风险低
🟡 焦虑型：极度依赖回复，高频稳定是关键
🔵 回避型：潜水不代表无问题，需主动破冰
🔴 混乱型：情绪周期大，断崖式流失风险高
💡
决策风格分析
识别关单引爆点和致命禁忌——
💹 价值驱动：看ROI，给蓝图
❤️ 关系驱动：信任第一，先建关系
🛡️ 风险规避：怕亏损，给保障承诺
👥 从众参照：给同类人成功案例

04 / SYSTEM

自动化解码引擎架构

解码引擎完全基于 Python 标准库构建，无额外依赖，可在服务器直接运行。支持缓存复用，同一用户的聊天记录只需从企微 API 拉取一次。

📡
数据抓取层
调用企微 API 拉取私聊+群聊记录，按时间轴清洗融合，输出结构化聊天流。支持最长 400 天历史数据。
💾
缓存管理层
首次拉取后自动落盘至 xiboData/聊天记录/UID_xxx_聊天记录.txt，重复运行直接命中缓存，零 API 调用。
🧠
AI 解码层
将 SKILL.md v3.0（九大板块）+ 聊天原石拼装为 Prompt，调用 GPT-4o / Gemini 进行深层心理解码，输出结构化报告。
📂
报告落盘层
标准化报告存储至 xiboData/用户画像数据/UID_xxx_分析报告.md，遵循统一命名规范，便于批量检索。
📌 快速命令参考
首次分析 python3 analyze_wecom_chat.py <UID>
强制刷新 python3 analyze_wecom_chat.py <UID> --refresh
缓存目录 xiboData/聊天记录/UID_xxx_聊天记录.txt
