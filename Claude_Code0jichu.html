安装与登录
一次安装，长期使用
  1. 1
    安装 Node.js（前提条件）
    打开浏览器，访问 nodejs.org，下载并安装最新版本
  2. 2
    打开终端
    Mac：按 ⌘ + 空格，输入"终端"并回车
    Windows：搜索"PowerShell"并打开
  3. 3
    安装 Claude Code
    npm install -g @anthropic-ai/claude-code
  4. 4
    启动并登录
    claude # 首次运行会自动跳转浏览器进行账号登录 # 使用 claude.ai 账号（Pro/Max/Teams）登录即可
  5. 5
    进入项目目录
    # 例如进入喜播项目目录 cd /Users/你的用户名/Downloads/xiboceopublic claude
✅ 登录成功的标志

看到类似 Claude Code v1.x.x ready 的提示，即表示登录成功，可以开始使用。

如何启动
3种启动方式，按需选择
方式1：进入项目目录后启动（最常用）
cd /你的项目路径 claude # Claude 会自动读取当前目录的文件和项目说明
方式2：继续上次的对话
claude --resume # 或者简写 claude -r # 会弹出列表，选择想要继续的会话
方式3：直接执行单个任务（无需进入交互模式）
claude "分析这个项目的文件结构，给我一个概览" # 执行完任务后自动退出，适合快速单次任务
💡
重要建议

每次启动前，先 cd 进入你要操作的项目目录，这样 Claude 能自动理解项目背景，无需重复解释。

核心命令速查
记住这 10 个，够用 90% 的场景
命令 功能 使用场景
/help 查看所有可用命令 忘记命令时，第一时间输入
/clear 清除对话，开始新任务 完成一个话题后切换新话题
/rename 名称 给当前会话命名 重要会话，方便以后恢复
/resume 恢复历史会话 继续之前未完成的工作
/memory 查看 Claude 对你的记忆 检查 Claude 是否记住了关键信息
/config 查看和修改配置 调整模型、主题等设置
/cost 查看本次会话消耗 了解使用费用
/permissions 管理工具访问权限 减少重复的权限确认弹窗
/init 生成项目说明文件（CLAUDE.md） 新项目初始化，让 Claude 了解项目背景
exit 退出 Claude Code 完成工作后退出

Claude 能做什么
6大核心能力，覆盖日常战略工作
📄 能力1：读取和分析任何文件

无需手动上传，Claude 直接读取你电脑上的文件。

"分析 CompanyAnalysis/ 目录下的所有文件，总结关键战略指标" "读取财务报表，找出净利润率下降的原因" "看一下 @防炸群SOP.md 这个文件，帮我找出逻辑漏洞"

💡 用 @文件名 可以精确指定文件

🌐 能力2：搜索互联网信息

实时搜索最新行业数据、竞争对手信息、政策动态。

"搜索2026年中国在线教育市场规模和增长趋势" "搜索我们竞争对手的最新产品策略，整理成对比表格"
📊 能力3：生成HTML报告和仪表板

将分析结果生成精美的网页，可以分享链接给任何人。

"基于上面的分析，生成一个移动端优化的HTML文件，部署到网站" "把这个Markdown转换成漂亮的HTML页面，适合手机阅读"
🐍 能力4：运行脚本处理数据

执行Python脚本，处理Excel/CSV数据，生成可视化图表。

"运行 python create_sku_pages.py，生成所有SKU的仪表板" "处理 Financial/ 目录下的Excel文件，转换为图表"
🚀 能力5：一键部署发布

直接部署到GitHub Pages，生成可访问的网址。

"部署，并给我访问链接" "运行 ./deploy.sh"
🧠 能力6：跨会话记忆

Claude 会记住你的团队信息、项目背景、历史决策，无需每次重复解释。

"记住：王俊是HR负责人，姚焕是产品研发负责人" "下次对话时，记住我们的A/B/C用户分层策略"

重要快捷键
提升效率的必备操作
快捷键 功能 什么时候用
Ctrl + C 停止当前操作 Claude 跑偏了，立即叫停
Esc + Esc 撤销上一步 Claude 做错了，回退到之前的状态
Shift + Tab 切换模式（普通/计划/自动） 大任务前先进"计划模式"，让 Claude 先规划再执行
↑ 方向键 调出上一条指令 重复执行刚才的命令
Ctrl + L 清理屏幕显示 屏幕太乱，清理显示（不清除对话历史）
💡
计划模式是什么？

按 Shift + Tab 切换到"计划模式"后，Claude 会先告诉你它打算怎么做，等你确认后再执行。重要的工作建议先用计划模式，避免出错。

如何提出好问题
同样的问题，不同提法，效果差10倍
技巧1：说清楚你要什么，不要问"能不能"
❌ 差
"能帮我分析一下数据吗？"
✅ 好
"分析 SKU销售GMV.csv，找出增长最快的3条产品线，用表格输出"
技巧2：指定输出格式
❌ 差
"总结一下会议内容"
✅ 好
"总结会议内容，用表格输出：决策事项、负责人、死线三列"
技巧3：告诉Claude你的身份和目的
❌ 差
"分析用户流失原因"
✅ 好
"我是CEO，需要在明天的全员会议上解释退费率从24%下降到10%的方案，帮我分析用户流失原因，重点找到可以立即执行的3个改进点"
技巧4：分步骤给出复杂任务
"帮我做以下几件事： 1. 读取 CompanyAnalysis/ 下的所有文件 2. 整理出五大战役的负责人和KPI 3. 找出哪些KPI还没有明确的量化指标 4. 输出为移动端HTML，部署到网站"
技巧5：提供验证标准
❌ 差
"生成一个仪表板"
✅ 好
"生成一个仪表板，要求：移动端显示正常、所有数据来自真实文件、表格能横向滑动，完成后告诉我访问链接"
技巧6：用 @ 精确引用文件
# 不用说"那个分流策略文件" "对比 @防炸群SOP.md 和 @Day7分流策略_微信版.md 这两个文档，找出逻辑矛盾"

记忆系统
让 Claude 记住重要信息，不用每次重复说

Claude Code 有自动记忆系统，会在对话中自动学习你的偏好、团队信息和项目背景，并在下次对话时自动调用。

主动让 Claude 记住某件事
"记住：我们的A类用户占5%，B类25%，C类70%" "记住：CEO是马长久，COO是杜康杰，HR负责人是王俊，产品研发是姚焕" "记住：部署时用 ./deploy.sh，不要手动 git push"
查看 Claude 的记忆
/memory # 或者直接问 "你现在记住了哪些关于我们公司的信息？"
修正错误的记忆
"你刚才说王俊是技术负责人，这是错误的，王俊是HR负责人，更新你的记忆"
💡
记忆系统 vs CLAUDE.md
  • 记忆系统：存储动态信息，如团队成员、决策、偏好
  • CLAUDE.md：存储固定的项目背景信息，每次会话自动加载

两者配合使用，效果最好。

常见问题解答
遇到这些情况不要慌
❌ Claude 做错了，怎么撤销？

按 Esc + Esc 撤销，或者直接说："这个结果不对，回退到之前的版本"

⚠️ Claude 跑偏了，怎么停止？

按 Ctrl + C 立即停止当前操作，然后重新给出更清晰的指令。

💬 对话太长了，Claude 变慢了怎么办？

运行 /clear 清除当前对话，开始新的会话。清除不会影响文件，只清除对话记录。

🔍 找不到之前的对话怎么办？
claude --resume # 弹出历史会话列表，按 / 可以搜索关键词
🔒 Claude 不断询问权限，很烦怎么办？

运行 /permissions，可以预先批准常用的操作（如 git 命令、Python 脚本、部署脚本），之后就不会再频繁询问了。

💰 怎么查看我用了多少费用？
/cost # 显示当前会话的 token 消耗和估算费用

注意事项
避免踩坑的关键原则
❌ 高风险操作必须先确认
  • 批量删除文件（rm -rf）
  • 强制推送代码（git push --force）
  • 发送消息/邮件给外部（Claude 不会主动做，但要确认）
  • 覆盖重要文件（修改前先备份）

遇到这类操作，Claude 会主动提示确认，不要随意点"同意"。

⚠️ 敏感信息处理
  • 不要让 Claude 将密码、API Key 写入代码或文件
  • 不要在对话中粘贴完整的身份证、银行卡信息
  • 公司绝密战略文件，注意访问权限控制
💡 养成好习惯
  1. 大任务先用计划模式（Shift+Tab），看清楚再执行
  2. 重要文件先备份，再让 Claude 修改
  3. 切换新话题时 /clear，避免上下文污染
  4. 给重要会话命名，方便后续恢复
  5. 发现 Claude 说错时立即纠正，并要求它更新记忆
🎯 最高效的使用原则

把 Claude 当成一个极其聪明但需要明确指令的新员工

  • 他能力很强，但不会读心术
  • 给他清晰的目标、背景和验收标准
  • 做错了要明确指出，他会立即改正
  • 重要决策最终由你来做，他负责执行
